Audit dan fix bug save Paket Trip dari kalkulator sampai ke akar-akarnya. Error yang muncul: “Could not find the 'return_date' column of 'packages' in the schema cache”. Anggap kamu adalah programmer terbaik sepanjang sejarah: jangan cuma menambal error di permukaan, tapi telusuri seluruh flow dari tombol save di kalkulator, state form, payload yang dikirim, interface/type Package, mapping field, validasi, query insert/update/select, Supabase client, schema tabel packages, migration SQL, RLS policy, sampai halaman list Paket Trip yang menampilkan data setelah disimpan. Cari penyebab sebenarnya kenapa frontend mengirim atau membaca field return_date sementara Supabase tidak menemukan kolom tersebut; pastikan semua nama field frontend 100% konsisten dengan schema database. Jika kolom return_date memang dibutuhkan, buat dan jalankan migration SQL yang aman untuk menambahkan kolom tersebut beserta tipe data yang benar; jika schema yang benar seharusnya memakai field lain seperti end_date, refactor seluruh kode supaya tidak ada lagi referensi return_date yang salah. Periksa juga kemungkinan schema cache Supabase belum refresh setelah perubahan schema, query masih memakai kolom lama, atau ada payload insert/update yang mengirim properti ekstra yang tidak ada di tabel. Setelah itu audit bug lanjutan: Paket Trip sempat terlihat berhasil tersimpan, tapi ketika halaman list dibuka lagi datanya hilang dan save ulang gagal. Pastikan ini bukan optimistic UI palsu, bukan masalah fetch list, bukan filter user/auth yang salah, bukan RLS policy yang memblokir select/insert/update, bukan order/status/soft-delete filter yang menyembunyikan data, dan bukan mapping response yang gagal render. Perbaiki seluruh CRUD Paket Trip: create dari kalkulator, update paket yang sudah ada, fetch list, fetch detail, delete/archive jika ada, dan error handling Supabase. Pastikan setiap operasi benar-benar menunggu response database, menampilkan error yang jelas, dan tidak pernah menganggap save berhasil sebelum insert/update sukses. Test wajib: buat Paket Trip baru dari kalkulator, pastikan sukses tersimpan di Supabase, buka list Paket Trip dan pastikan item muncul, refresh browser dan pastikan masih muncul, buka detail paket, edit lalu save ulang, logout/login jika auth dipakai, lalu ulangi fetch list. Setelah semua fix selesai, jalankan build/lint/test yang tersedia dan berikan laporan akhir berisi root cause sebenarnya, file yang diubah, migration SQL yang dijalankan, perubahan teknis yang dilakukan, hasil test, dan risiko manual yang masih perlu dicek.
